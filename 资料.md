# 足彩预测辅助系统 v2.0 — 完整开发说明文档

---

## 修订说明（v1 → v2）

本次修订基于GPT、Deepseek、Grok三家对v1文档的评审意见，主要修复以下问题：

1. 平局亚盘信号映射逻辑未闭环 → 新增平局专用确认规则
2. 体彩抽水率表述错误（65%→正确约32%）→ 全文修正
3. 平局修正条件C存在循环论证（亚盘数据在模块一时尚未输入）→ 删除条件C，改为仅用A+B
4. 双重确认"独立性"问题 → 明确两套信号的分工定位，亚盘改为"过滤器"而非第二预测器
5. 示例中EV与亚盘方向矛盾 → 修正示例逻辑
6. JSON记录缺少原始输入字段 → 补充veto_model_inputs字段
7. 安全边际阈值过低 → 提升EV触发标准，增加真实概率差值判断

## 项目定位（重要，请先阅读）

**这个系统是什么：** 把主观读盘直觉转化为结构化记录的辅助分析工具。

**这个系统不是什么：** 保证长期盈利的量化黑箱。

**核心设计思路：**
- 信号A（否决模型）：给出基于近期战绩的真实概率基准
- 信号B（亚盘框架）：作为过滤器，判断"这场是否值得信任模型的输出"
- 两者分工明确，A负责概率，B负责过滤，不重复计算市场信息

**关于体彩抽水的现实：**
竞彩足球官方返奖比例约68%（含1%调节基金），即抽水约32%。这意味着即使模型EV > 1.0，实际长期期望仍可能为负。系统的价值在于找到相对高EV且亚盘信号支持的场次，而非保证盈利。建议以小资金严格执行，把前100场当作数据积累期，而非盈利期。

## 模块一：否决模型（Veto Model）

### 前置输入（流程开始时收集）

在进入主流程之前，程序先询问：
```
请输入亚盘终盘主水和客水（用于平局修正判断）
主水：___  客水：___
（不知道可输入0跳过，修正条件C将不触发）
```

### 主要输入参数

- 主队近N场结果序列，例如：W D L W W D W L W W
- 客队近N场结果序列
- N值（默认10，可调整为5-15）
- 衰减因子decay_factor（默认0.8，范围0.5-1.0）
- 联赛选择（用于获取平局率参考值）

### 算法步骤

**第一步：计算各队加权历史概率**

```
对于最近第i场比赛，权重 w_i = decay_factor^(N-i)

P_H(win)  = Σ(w_i × 1[胜]) / Σ(w_i)
P_H(draw) = Σ(w_i × 1[平]) / Σ(w_i)
P_H(loss) = Σ(w_i × 1[负]) / Σ(w_i)

客队同理计算 P_A(win), P_A(draw), P_A(loss)
```

**第二步：交锋初始概率（否决逻辑）**

```
P(主胜) = P_H(win) × (1 - P_A(win))
P(客胜) = P_A(win) × (1 - P_H(win))
P(平局) = (P_H(draw) + P_A(draw)) / 2
```

**第三步：归一化**

```
S = P(主胜) + P(平局) + P(客胜)
P'(主胜) = P(主胜) / S
P'(平局) = P(平局) / S
P'(客胜) = P(客胜) / S
```

### 平局修正

触发条件：满足以下任意两条

- 条件A：该联赛历史平局率 > 28%
- 条件B：主队近5场平局数 + 客队近5场平局数 ≥ 4场
- 条件C：亚盘终盘双方水位差 < 0.05（需前置输入数据，若用户跳过则此条不可用）

触发后：
```
P'(平局)修正 = P'(平局) × 1.15
重新归一化三项使总和=1
```

### 联赛平局率参考表（内置）

```
J联赛    ：30%
意甲     ：28%
荷甲     ：27%
英超/西甲：25%
德甲/中超：24%
澳超     ：22%
其他/未知：25%（默认）
```

## 模块二：体彩欧赔转换

### 输入

- 体彩主胜赔率 O_home
- 体彩平局赔率 O_draw
- 体彩客胜赔率 O_away

### 计算

```
返还率基准：μ = 1 / (1/O_home + 1/O_draw + 1/O_away)

标准化隐含概率：
P_implied(主胜) = μ / O_home
P_implied(平局) = μ / O_draw
P_implied(客胜) = μ / O_away

体彩抽水率（显示用）：
抽水率 = (1/O_home + 1/O_draw + 1/O_away - 1) × 100%
```

注：竞彩官方规定返奖约68%，抽水约32%。若计算结果与此差异较大，可能是赔率输入有误。

## 模块三：价值识别

### EV计算

```
EV(主胜) = P'(主胜) × O_home
EV(平局) = P'(平局) × O_draw
EV(客胜) = P'(客胜) × O_away
```

### 安全边际判断（v2新增）

仅EV > 1.0不够，还需满足概率差值条件：

```
差值(主胜) = P'(主胜) - P_implied(主胜)
差值(平局) = P'(平局) - P_implied(平局)
差值(客胜) = P'(客胜) - P_implied(客胜)
```

**有效价值投注的双重条件：**
```
EV > 1.05（基础门槛）
AND
差值 > 0.06（真实概率比隐含概率高6个百分点以上）
```

两个条件都满足，才标记为"有效价值"。仅满足一个，标记为"弱价值，谨慎"。

### 输出示例

```
EV分析：
主胜：P'=38.2%  隐含=46.5%  差值=-8.3%  EV=0.80  ✗ 无价值
平局：P'=33.5%  隐含=30.5%  差值=+3.0%  EV=1.08  △ 弱价值（差值不足6%）
客胜：P'=28.3%  隐含=23.0%  差值=+5.3%  EV=0.99  ✗ 无价值
```

## 模块四：亚盘五层信号评分

每个信号由用户根据实际盘口观察打分。正值方向=支持主队，负值方向=支持客队。

**评分范围：每个信号 +1 / 0 / -1，总分 -5 到 +5**

---

### 信号1：平博重心方向

```
平博终盘主水 < 客水（差值 > 0.03）→ 偏主 → +1
平博终盘主水 ≈ 客水（差值 ≤ 0.03）→ 中性 → 0
平博终盘主水 > 客水（差值 > 0.03）→ 偏客 → -1
```

---

### 信号2：平博 vs 威廉希尔分歧

```
两家初盘方向相反（各自偏向不同队，水位差均 > 0.10）→ 强分歧
  → 跟随平博方向：平博偏主则+1，平博偏客则-1

两家初盘方向一致 → 无分歧 → 0
分歧存在但幅度小（水位差 < 0.10）→ 弱分歧 → 0
```

---

### 信号3：终盘水位绝对差

```
终盘主水比客水低超过0.08 → 资金明显偏主 → +1
终盘客水比主水低超过0.08 → 资金明显偏客 → -1
差值在0.08以内 → 市场均衡 → 0
```

---

### 信号4：盘口与水位背离

```
盘口升（向主队方向移动），但主水未降反升 → 庄家不跟资金 → 偏客 → -1
盘口降（向客队方向移动），但客水未降反升 → 庄家不跟资金 → 偏主 → +1
盘口变动与水位方向一致 → 无背离 → 0
盘口无变化 → 0
```

---

### 信号5：降盘异常

```
赛前出现明显降盘后回升（跨越0.5个让球档位以上）→ 有知情资金试探
  → 回升方向偏主：+1
  → 回升方向偏客：-1
无异常 → 0
```

---

### 模块四输出

```
S1 平博重心：X
S2 公司分歧：X
S3 水位差值：X
S4 盘口背离：X
S5 降盘异常：X
─────────────
亚盘总评分：X分

解读：
+4到+5：亚盘强烈支持主队，信号高度可信
+2到+3：亚盘偏主，信号较清晰
+1：亚盘轻微偏主，信号偏弱
0：亚盘中性，无明确方向
-1：亚盘轻微偏客，信号偏弱
-2到-3：亚盘偏客，信号较清晰
-4到-5：亚盘强烈支持客队，信号高度可信
```

## 模块五：双重确认决策引擎（v2核心修订）

### 设计原则（v2修订）

亚盘信号在本模块中扮演"过滤器"角色，而非第二预测器：
- 模块三负责：这场有没有数学价值
- 模块四负责：这场的市场信号是否支持下注

亚盘总分的方向与模型推荐选项的对应关系：
```
推荐主胜 → 亚盘总分需为正（+1以上）才确认
推荐客胜 → 亚盘总分需为负（-1以下）才确认
推荐平局 → 亚盘总分需在-1到+1之间（市场无明确倾向）才确认
           （平局对应"市场平衡"，而非偏向某一方）
```

### 决策规则

**规则1：强烈建议**
```
条件：
  模块三某选项满足双重条件（EV>1.05 AND 差值>6%）
  AND
  亚盘信号方向与该选项一致（|总分|≥2 且方向匹配）

输出：★★★ 强烈建议 [选项]
      EV=[数值] / 概率差=[数值] / 亚盘评分=[分数]
```

**规则2：建议考虑**
```
条件：
  模块三某选项EV>1.05 但差值在3%-6%之间（弱价值）
  AND 亚盘信号方向一致（|总分|≥1）

  或

  模块三满足双重条件
  AND 亚盘信号方向一致但强度偏弱（|总分|=1）

输出：★★ 建议考虑 [选项]，信号偏弱，轻仓
```

**规则3：信号冲突，放弃**
```
条件：模块三有有效价值选项，但亚盘方向相反

输出：❌ 信号冲突，不建议下注
      模型看好[X]，但亚盘指向[Y]，两者矛盾
```

**规则4：仅亚盘有信号**
```
条件：模块三无有效价值，但亚盘|总分|≥3

输出：⚠️ 无数学价值，亚盘信号强（[分数]分）
      记录观察，本场不建议下注
```

**规则5：无信号**
```
条件：以上均不满足

输出：— 本场无有效信号，跳过
```

### 平局专项规则（v2新增）

当模块三中EV最高选项为平局时，使用以下独立逻辑：

```
平局双重确认条件：
  EV(平局) > 1.05 AND 差值 > 6%
  AND 亚盘总分在 -1 到 +1 之间

→ ★★★ 强烈建议：平局（市场无明确倾向，与模型吻合）

平局弱确认条件：
  EV(平局) > 1.05
  AND 亚盘总分在 -2 到 +2 之间

→ ★★ 建议考虑：平局

平局信号冲突：
  EV(平局)最高
  AND 亚盘|总分| ≥ 3（市场明确偏向一方）

→ ❌ 平局价值存在，但市场明显偏向[主/客]，信号冲突，不建议
```

    "away_sequence": "D W W L W D W W D L",
    "n": 10,

    "home": 46.5,
    "draw": 30.5,
    "away": 23.0
  },
  "ev": {
    "home": 0.80,
    "draw": 1.08,
    "away": 0.99
  },
  "probability_gap": {
    "home": -8.3,
    "draw": 3.0,
    "away": 5.3
  },
  "asian_signals": {
    "s1_pinnacle_lean": -1,

- `model_version`：记录当时使用的模型版本，防止调参后污染历史数据

## 已知局限性（供使用者知悉）

1. **否决模型的粗糙性**：仅基于近期W/D/L序列，不区分对手强弱、主客场、伤停，在高强度联赛中可能偏差较大。这是第一期的有意简化，第二期可引入主客场分拆或对手强度加权。

2. **亚盘信号的主观性**：五层信号依赖用户手动打分，不同时间、不同状态下评分可能不一致。100场记录后，可通过统计找出哪些信号实际上对你来说是可靠的。

3. **体彩高抽水的现实**：竞彩抽水约32%，意味着即使EV > 1.05，长期期望仍可能偏负。本系统定位是"纪律化分析工具"，不是盈利保证。

4. **双重信号的相关性**：亚盘信号与体彩欧赔同属市场价格生态，不是完全独立的两套系统。亚盘在本框架中扮演"过滤器"而非"独立预测器"，这是设计上的有意取舍。

5. **样本偏差**：用户往往只分析自己有把握的场次，导致记录样本可能不具有代表性。建议在统计分析时注意这一点。
