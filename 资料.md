九、请实现以下核心规则组
========================

【规则组 A：体彩 vs 欧赔差异识别】

目标：
识别体彩和欧赔终赔之间的概率偏差。

建议逻辑：
- 如果体彩某一项去水概率显著低于欧赔终赔该项概率，说明体彩该方向赔率相对更“甜”
- 这不是直接下注信号，但可以作为加分项
- 阈值写入 config

请实现：
- diff_home
- diff_draw
- diff_away
并根据阈值加减分

【规则组 B：欧赔压低识别】

目标：
识别市场对某个结果的强化。

建议逻辑：
- 如果主胜终赔较初赔明显下调，主胜加分
- 如果平赔下调，平局加分
- 如果客胜下调，客胜加分
- 变化幅度越大，加分越高
- 阈值可配置

【规则组 C：亚盘强化识别】

目标：
识别让球市场支持方向。

建议逻辑：
- 如果主队从更浅盘走到更深盘，或同盘口持续降水，则主队加分
- 如果客队盘口方向增强，则客队加分
- 盘口变化比水位变化权重更高
- 若盘口不动但持续降水，也可给弱加分

【规则组 D：盘口赔率一致性】

目标：
识别不同市场是否讲同一件事。

建议逻辑：
- 欧赔主胜下调 + 亚盘主队升盘/降水 => 主队强一致信号
- 欧赔客胜下调 + 亚盘客队方向增强 => 客队强一致信号
- 欧赔平赔下调 + 亚盘无明显强化某方 => 平局增强
- 一致性高 => 强加分
- 不一致 => 提高风险等级

【规则组 E：赔率强但盘口弱 / 盘口强但赔率弱】

目标：
识别表面强势但结构不确认的比赛。

建议逻辑：
- 欧赔支持主队，但亚盘不确认甚至反向 => 主队减分
- 亚盘支持主队，但欧赔不确认 => 主队弱减分或风险提升
- 任一方向出现明显不一致，都应降低下注意愿

【规则组 F：热门过热】

目标：
避免追过热热门。

建议逻辑：
- 低赔热门如果继续被压低，同时盘口已经很深，则不应简单继续加分
- 应触发热门过热检查
- 如果赔率已经没有价值，输出“方向可能对，但赔率价值不足，建议放弃”
- 这是系统重要过滤器

【规则组 G：试探失败】

目标：
识别中途强化但最终回撤的比赛。

建议逻辑：
- 如果亚盘中途曾强化某方向，但终盘回到原盘或明显回拉
- 且欧赔未继续确认
- 记为试探失败
- 对应方向减分
- 风险等级提高

【规则组 H：临场确认】

目标：
识别临场市场最终确认的方向。

建议逻辑：
- 如果终盘相较中盘进一步强化同一方向
- 且欧赔同步确认
- 记为临场确认
- 该方向加分

【规则组 I：来回摇摆 / 高噪音】

目标：
识别看不懂、不该碰的比赛。

建议逻辑：
- 若盘口或水位多次反向
- 欧赔和亚盘一会儿一致一会儿不一致
- 记为高噪音结构
- 比赛总质量评分下降
- 默认建议放弃

【规则组 J：平局增强】

目标：
单独识别适合考虑平局或放弃的结构。

建议逻辑：
- 平赔下降且主客两边都没有形成一致强化
- 盘口不愿意明显强化任一方
- 方向互相打架时，如果平赔持续下调，可给平局加分

   - 热门方向是否存在赢球但不穿盘风险
4. 大小球盘口请作为第二阶段增强模块设计，先预留字段和规则接口，但不要让第一版过重
5. 第一版优先做：

   - 体彩让球胜平负

   - 亚盘
   - 路径结构
